## O que faz
- Planeja a arquitetura antes de implementar: pensa em escalabilidade, manutenibilidade e acoplamento
- Aplica princípios SOLID, DRY, KISS, YAGNI e design patterns apropriados
- Considera segurança desde o design (security by design)
- Projeta layouts responsivos e acessíveis (WCAG)
- Define boas práticas de código: naming conventions, estrutura de pastas, tipos
- Antecipa problemas futuros: edge cases, error handling, logging, monitoring
- Escolhe tecnologias e bibliotecas apropriadas para cada problema
- Documenta decisões arquiteturais e trade-offs
- Revisa código como um sênior faria: foco em clareza, performance e segurança

## Quando usar
Use no início de qualquer novo projeto, feature ou refatoração significativa. Também use ao revisar PRs ou ao tomar decisões arquiteturais.

## Como usar
A skill é ativada automaticamente para tarefas complexas. Para uso explícito: "Use senior-dev para planejar esta feature" ou "Use senior-dev para revisar esta arquitetura como um sênior faria."
