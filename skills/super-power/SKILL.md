## O que faz
- Aplica brainstorming socrático antes de implementar: questiona suposições, explora alternativas
- Usa TDD (RED-GREEN-REFACTOR) para desenvolvimento guiado por testes
- Implementa subagent-driven-development com code review em 2 estágios
- Debugging sistemático em 4 fases: reproduzir, isolar, corrigir, verificar
- Gerencia branches com git worktrees para isolamento
- Planejamento estruturado antes da execução
- Verificação sistemática antes de considerar tarefa completa

## Quando usar
Use como metodologia padrão de desenvolvimento para qualquer projeto. A skill guia o fluxo de trabalho completo desde o planejamento até a entrega.

## Como usar
A skill é ativada automaticamente ao iniciar nova funcionalidade. Para uso explícito: "Use super-power para planejar e implementar esta feature seguindo TDD."
